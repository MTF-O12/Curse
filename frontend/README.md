1. Описание проекта
Данный проект представляет собой личный портфолио-сайт, разработанный для демонстрации навыков, образования, интересов и контактной информации. Сайт включает полноценную систему приёма вакансий с хранением данных в базе MongoDB.
Проект выполнен в рамках учебной программы AITU и отражает реальный стек технологий, применяемый в современной веб-разработке.
1.1 Основной функционал
Отображение персональной информации: имя, возраст, город, контакты
Секция навыков с анимированными полосками прогресса
Блок образования с данными об университете AITU
Список языков (русский, английский)
Блок интересов (видеоигры, геймдизайн, ИИ в играх и др.)
Система вакансий: отправка, просмотр и удаление предложений
Все данные вакансий сохраняются в MongoDB через REST API


2. Установка и запуск
2.1 Требования
Node.js версии 18 и выше
npm (устанавливается вместе с Node.js)
Аккаунт MongoDB Atlas (бесплатно: cloud.mongodb.com)
2.2 Установка зависимостей
Фронтенд (React / Vite)
cd frontend
npm install
Бэкенд (Express)
cd backend
npm install express mongoose cors dotenv
2.3 Настройка переменных окружения
Создайте файл .env в папке backend со следующим содержимым:
MONGO_URI=mongodb+srv://ИМЯ:ПАРОЛЬ@cluster0.xxxxx.mongodb.net/vacancies
PORT=5000
Строку подключения MONGO_URI берёте в MongoDB Atlas: Clusters → Connect → Drivers.
2.4 Запуск проекта
Запуск бэкенда
cd backend
node server.js
В терминале должно появиться:
✅ MongoDB подключена
🚀 Сервер запущен на http://localhost:5000
Запуск фронтенда
cd frontend
npm run dev
Сайт откроется по адресу: http://localhost:5173

4. Документация API
Базовый URL: http://localhost:5000/api
4.1 GET /vacancies
Возвращает список всех вакансий, отсортированных по дате (новые первыми).
Параметры запроса
Нет параметров.
Пример ответа
[
  { "_id": "664abc...", "company": "Wargaming", "message": "Стажировка", "createdAt": "2025-06-01T10:00:00Z" }
]
HTTP коды
200 OK — успешно
500 Internal Server Error — ошибка сервера

4.2 POST /vacancies
Создаёт новую вакансию и сохраняет её в MongoDB.
Тело запроса (JSON)
Поле
Тип
Обязательно
Описание
company
string
Да
Название компании
message
string
Да
Текст сообщения/вакансии


Пример запроса
POST /api/vacancies
Content-Type: application/json


{ "company": "Wargaming", "message": "Ищем Junior Game Developer" }
Пример ответа
{ "success": true, "vacancy": { "_id": "664abc...", "company": "Wargaming", ... } }
HTTP коды
201 Created — вакансия создана
400 Bad Request — не заполнены обязательные поля
500 Internal Server Error — ошибка сервера

4.3 DELETE /vacancies/:id
Удаляет вакансию по её идентификатору MongoDB.
Параметры пути
id — строка ObjectId из MongoDB (например: 664abc123def456789012345)
Пример запроса
DELETE /api/vacancies/664abc123def456789012345
Пример ответа
{ "success": true }
HTTP коды
200 OK — удалено успешно
500 Internal Server Error — ошибка сервера

6. Руководство пользователя
6.1 Навигация по сайту
В верхней части страницы расположено липкое меню навигации. При клике на пункт меню страница плавно прокручивается к соответствующей секции. Доступные разделы:
О себе — краткое описание и статусы
Навыки — прогресс-бары и теги технологий
Образование — карточка AITU
Языки — родной и иностранный
Интересы — тематические пилюли
Вакансии — форма отправки предложений
6.2 Отправка вакансии
1. Прокрутите страницу до секции «Вакансии»
2. Введите название компании в первое поле
3. Введите сообщение или описание вакансии во второе поле
4. Нажмите кнопку «Отправить →»
5. При успехе появится зелёное сообщение «Вакансия отправлена и сохранена в MongoDB»
6. Вакансия сразу отобразится в списке ниже
6.3 Удаление вакансии
В каждой карточке вакансии находится кнопка «Удалить». При нажатии вакансия удаляется из базы данных и исчезает из списка без перезагрузки страницы.


7. Ключевые части кода
7.1 useEffect — наблюдатели прокрутки (App.jsx)
IntersectionObserver используется для анимации появления секций и полосок навыков при прокрутке. Функция cleanup (return) обязательна — без неё наблюдатели накапливаются при каждом рендере.
useEffect(() => {
  const obs = new IntersectionObserver(callback, { threshold: 0.08 });
  sections.forEach(s => obs.observe(s));
  return () => obs.disconnect(); // Очистка при размонтировании
}, []);
7.2 Компонент Vacancies.jsx
Компонент управляет тремя операциями: загрузкой списка (GET), отправкой формы (POST) и удалением (DELETE). Все состояния хранятся в React-стейте — без прямых манипуляций с DOM.
7.3 server.js — Express маршруты
Каждый маршрут обёрнут в try/catch для корректной обработки ошибок. Валидация входных данных выполняется до обращения к базе.
app.post('/api/vacancies', async (req, res) => {
  const { company, message } = req.body;
  if (!company || !message)
    return res.status(400).json({ error: 'Заполните все поля' });
  const vacancy = new Vacancy({ company, message });
  await vacancy.save();
  res.status(201).json({ success: true, vacancy });
});
